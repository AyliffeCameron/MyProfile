Project Idea

Overview

The Cryptocurrency Mining Equipment Reset System (CMERS) is designed as a low-cost DIY alternative to automate the hard reset of Crypto mining rigs in case of hardware error.

Partnered with commercial of the shelf (CoTS) hardware, this open-source software design is intuitive enough for casual miners yet provides professional level tools to manage your mining rig, no matter you experience level.

In addition to the automation, CMERS also provides web-based UI for remote functionality to rig mangers and system administrators alike. From here, they can send a hard reset command at the simple click of the button.

Motivation

2021 so far has seen a rise in value of the most popular cryptocurrency and with it so too has the profitability of several crypto algorithms (PCGAMERS).

While arguably the words most popular crypto, Bitcoin is still mined by large organisations with specific hardware, other cryptos such as Ethereum are again attractive to GPU miners. Serious gamers and DIY farmers are once again dragging out their old hardware chasing that elusive return on investment (ROI).

Graphics Processing Units (GPUs) are an essential piece of computer hardware for any serious gamer or graphic designer. However, GPUs are also especially handy for solving the complex mathematical problems that makes some cryptocurrencies secure.

Using multiple GPUs for mining purposes tends to result in unstable or unresponsive hardware and often results in a hard reset. While the casual miners often have a day job and a rig parked in the corner of a room, there is likely a delay in accessing it physically to hit the reset button which results in lost profit while the rig is down.

Description

CMERS uses a standalone computing hardware to act as a remote hard reset switch.

The reset switch for Personal Computer (PC) motherboards is generally connected to two pins on the motherboard with one being a ground pin. When the reset button is depressed on the PC case, the circuit is closed and such grounds out the second pin triggering the hard reset.

With a ground and a general purpose input/output pin of the SoC computer connected to a NPN Transistor we can trigger the Transistor to switch closed thus simulating a reset button press. The wiring schematic would be as shown below in Figure 1. The GPIO ground pin would connect to the collector terminal of the transistor and GPIO signal pin to the transistor base terminal.

The Emitter pin on the transistor would be connected to the sign pin on the mother board or alternatively splice into PC case reset button such that it may work in tandem with the PC cases reset button.

The Raspberry Pi will run a lite version of Ubuntu server as the base OS. The Apache webserver package is required to deliver the webUI. The web UI is to be coded in HTML to present the user with a button which will run the required script triggering the reset of the connected device.

Several python scripts will be required to be written to perform the control of the 3 volt GPIO pin and monitor the mining operation. These scripts will be run in a matter of two ways.

Firstly, once the user accesses the webUI, a single button with the text “RESET RIG” will be displayed. When pressed, this should run the python script setting the GPIO pin to high which will allow the transistor to complete the ground circuit, triggering the reset.

In addition to the above, a second script should be automatically run after system boot. The script will be run by a cron job for approximately 5 minutes after mining rig boot however the user should set this to a time after they can verify the rig is mining. This script’s purpose is to monitor mining operations by pinging the mining rig for activity. If the reply is “unreachable” the rig will be considered offline or frozen and trigger the above reset script otherwise snooze for another 5 minutes and then run again.

Future Expansion

As the demand in GPU use increases due to mining operations the computer will in turn draw more current. This may be exploited by the use power monitoring device such as the emonPi Shield KIT by open energy monitor to monitor power load data for changes. This data could be used to determine a “normal” operation power load. The auto reset script could than monitor this and when the power draw is below a set threshold, the reset script is triggered.

Tools and Technologies

CMERS uses a small System on a Chip (SoC) computer called a Raspberry Pi with General Purpose Input/Output (GPIO) pins. The Raspberry Pi is connected to the mining rigs mainboard reset pins which are used to trigger a hard reset of the system. For remote access to CMERS, the SoC computer must be network connected.

A headless version of Linux running a webserver such as APACHE and some simple scripts written in Python takes care of the software requirements.

A Single GPIO jumper wire is all that is required to connect CMERS to the mining rig.

  • NPN Transistor
  • Breadboard

Users of CMERS should be able to have it up and running in no time and be able to remotely reset their mining rigs from anywhere as simply as playing a video on a webpage.
